# Сопроводительное письмо

**Кандидат:** Даниил Рамкулов  
**Задание:** веб-инструмент с одним вопросом и обработкой ответа через LLM

---

## Решение по системному промпту

### Проблема «наивного» промпта

Формулировка из ТЗ — «если поверхностный — задай вопрос, если подробный — поблагодари» — корректна по сути, но оставляет модели слишком много свободы. На практике LLM часто:

- задаёт **несколько** вопросов сразу;
- «благодарит» поверхностные ответы из вежливости;
- использует закрытые вопросы («Вам понравился курс?»), которые плохо подходят для качественного интервью.

### Что я сделал

1. **Явная рубрика глубины** — в промпте перечислены конкретные маркеры «подробного» ответа (критерии выбора, источники информации, сравнение альтернатив, личный контекст, эмоции). Это снижает субъективность и делает поведение модели воспроизводимым.

2. **Жёсткий формат ответа** — ровно один из двух сценариев, без списков и без объяснения своей оценки респонденту. Так интерфейс остаётся простым: одно поле → один ответ исследователя.

3. **Требование к уточняющему вопросу** — только открытый вопрос, привязанный к уже сказанному. Это ближе к методологии глубинных интервью (laddering, probing).

4. **Низкая temperature (0.4)** — меньше «творчества», больше стабильности в исследовательской задаче.

5. **Groq (бесплатный LLM)** — реальная модель Llama 3.3 70B через Groq API: бесплатный tier без карты, OpenAI-compatible endpoint. Для тестового задания это оптимальный баланс «настоящий ИИ / нулевая стоимость / простая интеграция».

## Пояснение по коду

| Компонент | Зачем |
|-----------|-------|
| `prompts.py` | Промпт и рубрика отделены от логики — легко итерировать без правок API |
| `llm.py` | Один вход `analyze_answer()` → вызов Groq (Llama 3.3 70B) |
| `main.py` | Тонкий слой FastAPI: валидация, ошибки 502 от LLM, раздача фронтенда |
| `index.html` | Один файл, без сборки; аккуратный UI, счётчик символов, подсказка о коротком ответе |

Архитектура намеренно минимальная: это тестовое задание, а не микросервис. При этом код читается как заготовка для расширения (см. ниже).

## Что бы добавил в продукте

1. **Второй шаг диалога** — сейчас UI показывает уточняющий вопрос; в полноценном инструменте нужна цепочка сообщений (`messages[]`) и кнопка «Ответить на уточнение».

2. **Structured output** — просить модель возвращать JSON `{ "verdict": "shallow"|"detailed", "reply": "..." }` для надёжной логики UI и аналитики.

3. **Логирование интервью** — анонимное сохранение пар «ответ → реакция модели» для последующего кодирования исследователем.

4. **A/B промптов** — сравнение разных рубрик на одной выборке ответов.

5. **Rate limiting и auth** — если инструмент публичный.

---

## Почему это решение выделяется

- Промпт продуман как **исследовательский инструмент**, а не просто обёртка над ChatGPT.
- Подключена **реальная LLM** (Groq / Llama 3.3 70B) на бесплатном tier.
- UI аккуратный, но без лишней сложности (один HTML-файл, как в ТЗ).
- Тесты покрывают оба сценария (поверхностный / подробный).
- README и это письмо объясняют **почему**, а не только **как запустить**.
